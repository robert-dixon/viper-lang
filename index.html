Why Viper?

Viper is just another programming language. I didn't build it to solve some great unsolved programming problems - I built it because I wanted to build a language and this one is just the way I like it. Therefore, my personal taste and preference has been one of the main decisions in designing the language, and I like clean simple stuff. If this language just so happens to be your taste too, well then please join in!

I like being able to knock together proof-of-concept programs quickly and easily, and that's really what viper is designed for - for using at hackathons and the like, not for designing large serious systems. (But maybe one day...)

How Viper?

A set of tutorials will be published when I've got a little further with development. For now, to install on your machine, download

https://raw.githubusercontent.com/robert-dixon/viper-lang/master/viper-install.sh

and execute the following commands:

chmod +x ./viper-install.sh
. ./viper-install.sh

(Watch out - that first . is important, y'know)

You'll need code to look pretty while you're using Viper. If you use sublime, download

https://raw.githubusercontent.com/robert-dixon/viper-lang/master/viper.tmLanguage

And place it in your packages folder.
